01 · 项目概述

为劳动法而生
的智能法律咨询引擎

系统围绕劳动与社会保障法律领域，构建了从知识检索、推理计算到多轮对话的完整能力链路。 通过 RAG 保证回答的权威性与可追溯性，通过 ReAct Agent 实现复杂问题的多步骤拆解与工具协同。

权威知识问答

基于劳动法、劳动合同法、社保法等权威知识库提供即时回答，引用来源可追溯。

精确赔偿计算

支持经济补偿金(N)、违法解除赔偿金(2N)、加班费、未休年假补偿等四类常见计算。

ReAct 智能体

多步骤、多工具协同的复杂法律分析，深度思考模式让推理过程透明可见。

双交互模式

Streamlit Web 界面 + 命令行终端，多会话隔离，支持流式输出与文件管理。

modules/rag_module.py

RAGModule · 检索增强生成核心

整个系统最核心的模块，负责知识库加载、文档检索和基于 LLM 的问答生成。支持三种检索模式自由切换，并通过 RRF 融合算法兼顾准确率与召回率。

01
关键词检索
jieba 分词 + 自研评分算法（命中+3 / 子串+1 / 共现+0.5）
02
语义向量检索
Chroma + HuggingFace Embedding 余弦相似度
03
RRF 混合融合 DEFAULT
score(d) = Σ 1/(60+rank_i) · 综合排序
流式输出 联网增强 来源追溯 降级策略 指纹增量更新
# 三种检索模式自由切换 def _retrieve(self, query, top_k, mode): if mode == "keyword": return self._keyword_search(query, top_k) elif mode == "semantic": return self._semantic_search(query, top_k) else: # hybrid (RRF) kw = self._keyword_search(query, top_k) sm = self._semantic_search(query, top_k) return self._rrf_fuse(kw, sm, k=60) # 对外接口 ask_stream(question, session_id, mode) ask_with_web_search_stream(...)
modules/agent_module.py
~250 行

AgentModule · ReAct 智能体引擎

基于 LangGraph ReAct Agent 框架，提供多工具、多步骤的智能体推理能力。深度思考模式下，系统先通过 LLM 流式生成分析框架，再交由 Agent 执行。

PHASE 1 · THINKING
思考分析阶段
LLM 流式生成分析框架，输出 thinking 事件。6 步流程：明确问题 → 检索法条 → 交叉验证 → 精确计算 → 综合分析 → 引用来源。
PHASE 2 · REACT LOOP
Agent 执行阶段
Think → Act → Observe 循环，工具调用与 LLM 推理交替进行，直至生成最终答案。
STREAMING EVENTS
thinking 深度思考实时文本
tool_start 工具调用开始
tool_end 工具返回结果
answer_chunk 答案片段（token 级）
answer_complete 完整答案 + 写入历史

legal_calculator
StructuredTool · Pydantic

基于法律条款的精确赔偿计算器，支持 N / 2N / 加班费 / 未休年假四类计算。

日工资 = 月工资 / 21.75
LEGAL_CALCULATOR · 计算规则
类型 公式 法律依据
N · 经济补偿金 月工资 × 工作年限 《劳动合同法》
2N · 违法解除赔偿金 月工资 × 工作年限 × 2 《劳动合同法》第 87 条
overtime · 加班费 工作日 ×1.5 + 休息日 ×2 + 节假日 ×3 《劳动法》第 44 条
annual_leave · 未休年假 未休天数 × 日工资 × 3 《职工带薪年休假条例》第 5 条

UPLOAD
支持扩展

自定义知识文档

支持 PDF / DOCX / TXT / MD 格式，指纹增量更新避免重复构建。

01 · FINGERPRINT
指纹增量更新

基于文件路径、修改时间、分块参数的 MD5 哈希判断是否需要重建向量库，避免重复计算。

02 · AUTO RELOAD
自动重建索引

Web 界面上传新文件后自动触发知识库重载，并通过进程重启确保状态一致。

03 · FALLBACK
降级容错策略

Embedding 模型加载失败时自动降级为 keyword 模式，确保系统始终可用。
